FP&A 인턴 4개월차 · 정규직 전환까지 약 2개월

0. 한 줄 전략

회사에서의 나의 역할은?

FP&A 부서 내에서 단순 반복 작업을 자동화하여 업무 효율을 높여주는 오토메이션 가이.
통역이 아니라 "맡기면 결과물이 나오는 사람"으로 기억되게 한다.

1. 누가 (Who) — 나의 정체성 재정의

  • 반복 업무를 자동화·도구화해 팀의 시간을 아껴주는 FP&A 주니어
    • 팀원들에게 어떤 단순 반복작업이 있는지 먼저 파악 → 자동화 후보 발굴
  • 통역으로서는 부족하더라도, 업무에 지장이 없도록 필요한 영어/한국어 용어와 대화 스킬을 강화하는 사람
  • FP&A로서 알아야 할 지식을 하나씩 배워나가는 주니어

2. 누구에게 (To Whom) — 전환의 키를 쥔 사람

우선순위대상이유
1팀장 (평가권자)최종 평가권자이자 기대정렬의 핵심 상대
2가족일적으로도 정신적으로도 가장 든든하게 의지할 수 있는 존재
3팀 동료부족한 FP&A 지식과 실무를 배우고 도움받을 수 있는 상대
4문의 채널 사용자챗봇/트래커의 실제 "고객". 이들의 만족 = 임팩트 증거

3. 무엇을 (What) — 증명할 결과물

  1. 비용 문의 챗봇 — 문서 기반 답변 + 모를 때 에스컬레이션 루프, 완성
  2. Budget Tracker + 대시보드 — 분산 데이터 통합 → 매달 예산 집행 추적 → 이상 알림, 완성
  3. ★ 임팩트의 정량화 — 두 결과물이 만든 변화를 숫자로 (→ 8. 성과·KPI 설계)

4. 언제 (When) — 타임라인

  • 이번 주: 챗봇 완성 + Budget Tracker 대시보드 1차 + Before 베이스라인 측정
  • 다음 주: Budget Tracker 전체 완성
  • 그 이후 ~2개월: 임팩트 측정 → 팀장과 중간 점검 → 전환 어필 자료(1장 요약) 작성
  • ★ 지금 당장(이번 주 내): 팀장에게 "전환 기준이 무엇인지" 직접 묻는 1:1 — 기대정렬의 출발점

5. 어디서 (Where) — 가치를 보여주는 무대

  • 비용 문의 채널 — 챗봇이 실제로 도는 곳
  • 대시보드 — 팀장, 타 부서 리더, 경영진이 매달 들여다보는 곳(예상)
  • 주간 회의 / 1:1 — 진행상황과 임팩트를 직접 보고하는 곳

6. 왜 (Why) — 진짜 이유

  • 표면: 정규직 전환
  • 본질: 대체 불가능한 의존성 만들기.
    "이 사람한테 일을 맡기면, 챗봇·트래커처럼 만족스러운 결과물이 나온다"는 인식을 심어주기.

7. 어떻게 (How) — 기대정렬 실행법

  1. 기대 명확화 — 팀장에게 "정규직 전환을 위해 남은 기간 무엇을 보여드려야 하나요?"를 직접 질문. 추측 금지.
  2. 선제 보고 — 일정이 어긋나면 터지기 전에 "이 부분이 막혔고, X일까지 이렇게 하겠습니다"라고 먼저 말한다.
  3. 임팩트 정량화 — "만들었다"가 아니라 "이걸로 무엇이 얼마나 좋아졌다"로 보고한다.
  4. 강점 강화 — 자동화를 위해 개발을 이어가고, AI 활용을 꾸준히 공부한다.

8. 성과·KPI 설계

⚠️ KPI의 생명은 "도입 전(Before)" 숫자다.
챗봇·트래커가 자리잡으면 "원래 얼마나 걸렸는지"를 더는 잴 수 없다.
지금(이번 주) Before를 기록해두지 않으면, 2개월 뒤에 "좋아진 건 맞는데 증명은 못 하는" 상태가 된다.
또한 부풀린 숫자는 팀장에게 즉시 신뢰를 잃는다 → 방어 가능한 1~2개에 집중한다.

A. 비용 문의 챗봇

KPI측정 방법Before (도입 전)목표 (After)
자동 응답률사람 개입 없이 챗봇이 해결한 문의 비율0% (전부 수기 응답)70%+
평균 응답 대기시간문의 → 답변까지 걸린 시간평균 ○○시간즉시(초 단위)
담당자 시간 절감월 문의 건수 × 건당 처리시간월 ○○시간 소요월 ○○시간 회수
에스컬레이션율챗봇이 못 풀어 사람에게 넘어간 비율—시간이 갈수록 ↓ (학습 효과)
한 줄 성과 문장(전환 어필용):
"비용 문의 응답을 자동화해 담당자 월 ○○시간을 회수하고, 문의자 대기시간을 ○○시간 → 즉시로 줄였다."

B. Budget Tracker + 대시보드

KPI측정 방법Before (도입 전)목표 (After)
데이터 취합 시간여러 테이블 수기 통합에 든 시간월 ○○시간자동화(≈0)
예산 이상 조기 발견초과/이상 징후를 사전 감지한 건수월말 사후 발견월 ○건 사전 감지
현황 확인 주기예산 집행 현황을 확인하는 주기월 1회(월말)상시(실시간)
활용 범위대시보드를 보는 사람/부서 수0팀장 + 타 부서
한 줄 성과 문장(전환 어필용):
"분산돼 있던 예산 데이터를 자동 통합해 월 ○○시간의 수작업을 없애고, 예산 초과를 월말이 아닌 실시간으로 잡아냈다."

KPI를 만들기 위해 지금 해야 할 일

  1. Before 베이스라인 측정 — 지금 안 하면 영영 못 잰다. (수기 응답 시간, 월 문의 건수, 데이터 취합 시간 등)
  2. 자동 로깅 심기 — 챗봇 문의/응답/에스컬레이션 건수, 대시보드 접속·사용 기록을 자동으로 쌓이게 설계
  3. 주 단위로 숫자 수집 — 매주 같은 지표를 기록해 추세를 만든다
  4. 2주마다 "한 줄 성과 문장" 갱신 — 주간 회의/1:1에서 보고
  5. 전환 직전 1장짜리 성과 요약 — 위 숫자들을 모아 "내가 무엇을 했고, 무엇이 얼마나 좋아졌다"로 정리

9. 이번 주 체크리스트

  • 팀장과 1:1 잡기 — 전환 기준 직접 확인 (기대정렬)
  • Before 베이스라인 측정 — 챗봇/트래커 도입 전 숫자 기록 (최우선)
  • 챗봇 완성 + 채널에 정식 안내
  • Budget Tracker 대시보드 1차 공유
  • 챗봇/트래커에 자동 로깅 심기 (KPI 데이터가 저절로 쌓이게)
  • 챗봇/트래커 임팩트 측정 지표 확정 (무엇을 숫자로 보여줄지)
  • 주간 회의에서 진행상황 + 다음 주 계획 보고
